**Processo correto para qualquer mudança:**
1. Analisar o código e identificar os problemas
2. Listar o que será corrigido e pedir aprovação do Nilson
3. Executar todas as correções aprovadas
4. Deploy (se aplicável)
5. Pedir UM print no final para confirmar o resultado

**NÃO fazer:**
- Pedir print antes de cada mudança individual
- Fazer mudanças sem listar o que será feito primeiro
- Fazer mudanças sem aprovação
- Achismo — só mexer no que o código claramente indica como problema

**Why:** O Nilson tem o código nas mãos. Conseguimos identificar erros pelo código sem precisar de print a cada passo. Um print no final confirma o resultado. Pedir print demais é burocracia desnecessária.

**How to apply:** Para qualquer tarefa — analisa → lista os problemas → "posso fazer?" → executa tudo → deploya → pede UM print de confirmação.
